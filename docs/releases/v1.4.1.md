# ChatUI v1.4.1

## Fixes

- Restored the “回答中” indicator for non-active sessions that still have persisted chat jobs, image jobs, or pending submissions.
- On application startup and when the page returns to the foreground, resumes recoverable background-session tasks instead of only the active session.
- Keeps task recovery safe during session switching by relying on the existing per-session resume guard and handling the active session after its UI is rendered.
- Updated browser asset cache versions so deployments load the recovery fix immediately.

## Upgrade notes

- No configuration, API, or data migration is required.
- Tasks that have expired or no longer exist upstream will still finish with the existing recovery error state.
