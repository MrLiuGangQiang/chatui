# ChatUI v1.6.7

## Message order and history integrity

- Fixes blank or missing message indexes being interpreted as index zero, which could reorder conversations or remove messages during history compaction.
- Preserves every existing question-and-answer pair when loading older session snapshots with incomplete index metadata.
- Prevents pending recovery records with missing indexes from claiming or overwriting the first message in a conversation.

## Consistent edit and regenerate behavior

- Uses one canonical turn-replacement path for message editing, answer regeneration, and forced image generation.
- Ensures editing always replaces the selected question and its original answer, regardless of whether the current route is chat, image generation, or image editing.
- Creates a missing assistant replacement slot without overwriting the next user question.
- Persists the edit target and edited canonical history before routing so refresh and recovery do not append a second question-and-answer group.
- Reconciles duplicate recovery nodes back into the original answer position instead of displaying an extra answer at the end.

## Verification

- Added regression coverage for legacy blank indexes, pending-submit index recovery, canonical replacement slots, and duplicate DOM reconciliation.
- Verified normal edits, edits while image mode is active, and refresh during edit routing with browser-level recovery checks.
- Verified the complete project check suite with 326 passing tests.
- Docker deployments can upgrade to the `1.6.7` or `v1.6.7` image tag.
