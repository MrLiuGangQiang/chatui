# ChatUI v1.4.0

## Highlights

- Added a reproducible engineering baseline: editor and Git attributes, contribution guidance, Node 22 pinning, CI, and unified project checks.
- Formalized architecture, development, test, Docker, and release documentation.
- Moved the large test entry implementation into `test/legacy/` and kept `test/run-tests.js` as a small stable runner; added project-contract coverage.
- Hardened the release process: semantic tags now validate repository metadata and create a published GitHub Release after multi-architecture Docker images are pushed.
- Preserved the `route.html` static-page contract in the server, Docker image, checks, and tests to prevent route-diagram 404 regressions.

## Upgrade notes

- No application configuration or API contract changes are required for existing deployments.
- Use `npm run check` before contributing and `npm run verify:release -- vMAJOR.MINOR.PATCH` before creating a release tag.
