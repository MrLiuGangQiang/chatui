# ChatUI v1.5.3

## Reliability fixes

- Fixed an issue where one message could create two upstream completions when an accepted streaming request later encountered an SSE, rendering, or persistence error.
- Streaming fallback now creates a replacement request only when the original request failed before upstream acceptance and before any answer output began.

## Legacy session compatibility

- Restored automatic loading of conversations saved by versions before the IndexedDB session-snapshot migration.
- Legacy per-session messages, completed display records, pending tasks, and last-generated-image state are normalized and migrated into the current snapshot v2 format.
- Existing snapshot v2 data remains authoritative over stale local-storage backups, and old backups are retained during migration for recovery safety.

## Engineering

- Added regression coverage for accepted-stream retry suppression, legacy local-storage migration, version-one snapshot upgrades, and current-snapshot precedence.

## Upgrade notes

- No manual data conversion is required. Legacy conversations are migrated automatically when they are first loaded, provided the browser storage still exists under the same origin.
- Refresh the browser after upgrading so the updated workflow and session-persistence assets are loaded.
- Docker deployments can upgrade to the `1.5.3` or `v1.5.3` image tag.
