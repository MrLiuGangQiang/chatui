# ChatUI v1.7.1

## Highlights

- Resolves named historical images by persisted semantic metadata, so requests such as combining the cat and dog select those images even when many other image candidates exist.
- Asks for clarification only when the requested image cannot be mapped uniquely, including genuinely duplicate or underspecified subjects.
- Preserves image descriptions, prompts, labels, and revised prompts across generated-image persistence and history restoration.
- Restores selected images across multiple historical image results without silently falling back to a single image.
- Sends multi-image edit requests with repeated `image[]` multipart fields so every selected image reaches the upstream image edit API.

## Validation

- Full project checks and 356 automated tests.
- End-to-end semantic selection smoke test from conversation history through the real HTTP multipart request to a mock `/v1/images/edits` upstream.
- Headless Microsoft Edge (Chromium) browser verification using the built static bundle: four image candidates were loaded, only cat and dog were selected, and no page errors occurred.
