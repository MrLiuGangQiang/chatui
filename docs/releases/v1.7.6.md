# ChatUI v1.7.6

Released on July 20, 2026.

## Fixes

- Fixed intermittent Windows behavior where Enter inserted a newline instead of sending immediately after IME composition ended.
- Limited the post-composition trailing-Enter workaround to Apple platforms instead of applying the macOS Safari workaround globally.
- Preserved macOS IME candidate confirmation without accidental sending, while preventing the suppressed trailing Enter from inserting a newline.
- Consumed only one Apple trailing Enter so the next intentional Enter can send normally.

## Verification

- Added separate regression coverage for Windows, macOS, iPhone/iPad platform detection, active composition, legacy key code 229, and bound DOM event behavior.
- Verified in Microsoft Edge on Windows that Enter during composition does not submit and the immediate Enter after `compositionend` prevents the textarea newline and submits exactly once.
- `npm run check` passes with 371 tests.
