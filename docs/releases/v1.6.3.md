# ChatUI v1.6.3

## Durable task recovery

- Keeps a recoverable task owner from the instant Send is accepted through attachment capture, routing, managed-job handoff, and final session commit.
- Restores in-progress work after a page reload or session switch without starting duplicate requests.
- Runs GPT-5 Responses reasoning, Chat Completions, image generation, and image editing through the same server-managed job lifecycle.
- Rejects incomplete local job snapshots before contacting the upstream API, while matching a pending handoff only to the same complete job and submission identity.
- Commits final chat/image results and local clarification responses before clearing their recovery owner.

## Cancellation and failure safety

- Treats explicit Stop and session deletion as terminal, preventing late asynchronous work from recreating cancelled or deleted tasks.
- Preserves jobs across transient transport failures, but clears terminal failed jobs so they do not resume forever after every reload.
- Handles attachment-only submissions and reports when a reload occurred before attachment data could be durably captured instead of silently sending an incomplete request.

## Message actions

- Clears streaming and pending UI state synchronously when output completes, so copy, download, retry, and related message buttons no longer remain hidden when a background tab suspends animation frames.

## Verification

- Added lifecycle, handoff, reload, session-switch, cancellation, terminal-error, persistence-order, and message-action regressions.
- Verified the complete project check suite with 285 passing tests.
- Docker deployments can upgrade to the `1.6.3` or `v1.6.3` image tag.
