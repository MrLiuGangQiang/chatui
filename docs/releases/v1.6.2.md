# ChatUI v1.6.2

## Reasoning persistence hotfix

- Persists completed reasoning content with the canonical assistant message so it survives page refreshes.
- Restores saved reasoning together with the answer, even when reasoning is disabled for future requests.
- Keeps an in-flight request bound to the reasoning mode and effort selected in its originating session.

## Session-switch safety

- Prevents switching to a session with different reasoning settings from clearing another session's active reasoning output.
- Uses request-scoped reasoning settings for streaming, fallback requests, and GPT-5 Responses payloads.
- Keeps completed reasoning visible when the response settles after a session switch.

## Verification

- Added an end-to-end persistence regression covering request start, session switch, completion, snapshot storage, and full reload.
- Verified the complete project check suite with 268 passing tests.
- Docker deployments can upgrade to the `1.6.2` or `v1.6.2` image tag.
