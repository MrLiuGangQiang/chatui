# ChatUI v1.6.4

## Pending task continuity

- Shows a durable pending task immediately when a submission is accepted, including during routing and before the first upstream token arrives.
- Keeps pending chat and image tasks visible when switching away from a session and back, without creating duplicate placeholder messages.
- Rebinds live and recoverable jobs to their stable display identity so progress updates continue in the correct message position.

## Session rendering safety

- Separates canonical history validation from pending task projections, preventing legitimate pending messages from triggering unnecessary history repairs.
- Retains detached session DOM only for sessions that own active or durable work, while reconciling mutable progress details when the session becomes active again.
- Preserves media object URL ownership when cached session DOM is discarded.

## Verification

- Added regressions for pending-submit restoration, managed-job rebinding, route switching, canonical history repair, and stable session DOM caching.
- Verified the complete project check suite with 290 passing tests.
- Docker deployments can upgrade to the `1.6.4` or `v1.6.4` image tag.
