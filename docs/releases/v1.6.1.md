# ChatUI v1.6.1

## Session-switch hotfix

- Fixed session switching restarting an in-progress intent-recognition request.
- Prevents repeated switches from creating duplicate route, chat, or image requests for the same pending message.
- Keeps the original in-memory run as the sole execution owner until it completes or is stopped.

## Live task continuity

- Rebinds the intent-recognition and response placeholder to the currently rendered session node after switching back.
- Continues showing routing-stage progress instead of waiting for answer output before the task becomes visible again.
- Serializes durable pending-submit recovery after a refresh so only one recovery attempt can run per session.

## Message safety

- Prevents concurrent task completions from racing to overwrite the same response slot or session snapshot.
- Protects the latest user message from being lost during repeated session switching.
- Adds regression coverage for live-run ownership, single-flight recovery, stopped-task recovery, and post-switch node rebinding.

## Verification

- Verified the complete project check suite with 267 passing tests.
- Docker deployments can upgrade to the `1.6.1` or `v1.6.1` image tag.
