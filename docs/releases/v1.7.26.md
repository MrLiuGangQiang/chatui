# ChatUI v1.7.26

## Intent routing and historical visual references

- A webpage or HTML revision can now use a selected historical image as a visual style reference without being rejected as an invalid chat task.
- The selected historical image is restored from durable storage and included with the chat request, so the model receives the actual reference rather than only its identifier.
- Invalid intent-model output is now reported as a model-contract failure. It is no longer presented as though the user failed to state their goal, and it does not create a misleading pending clarification.

## Maintenance

- Removed an unused duplicate Markdown CommonJS resource loader. The browser continues to use its canonical dependency loader.

## Verification

- Ran `npm run check`: project, architecture, syntax, and all 441 automated tests passed.
