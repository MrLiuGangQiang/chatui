# ChatUI v1.6.5

## Reliable task completion state

- Centralizes chat, image, retry, stop, and recovery completion cleanup in a shared session task lifecycle.
- Resets the send button after a completed response even when recovery discovers an already-finished or already-cleared job snapshot.
- Preserves the busy state of a newer task when an older asynchronous completion arrives late.

## Browser audio behavior

- Creates or resumes the completion-sound `AudioContext` only during an active user gesture.
- Avoids browser autoplay-policy warnings when pending jobs resume automatically after startup or a session restore.
- Continues to play completion sounds when the audio context was previously unlocked by the user.

## Verification

- Added lifecycle regressions for normal completion, recovered completion, stale busy-state cleanup, and overlapping task ownership.
- Added completion-sound tests for browser user-activation behavior.
- Verified the complete project check suite with 295 passing tests.
- Docker deployments can upgrade to the `1.6.5` or `v1.6.5` image tag.
