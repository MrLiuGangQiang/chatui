# ChatUI v1.6.0

## Unified task routing

- Replaced the legacy mixed routing formats with one canonical `task_contract.v2` protocol for classification, review, clarification, and execution planning.
- Validates intent, task relationship, execution operation, resources, prompt plans, and clarification state before projecting them into the existing browser workflow.
- Rejects legacy route objects instead of silently adapting ambiguous model output.

## Context isolation

- Prevents a new image request from inheriting unrelated prompts or historical image resources from a previous task.
- Allows follow-up, correction, and continuation requests to preserve prior context only when the task contract explicitly identifies it.
- Keeps image-edit context fallback while separating it from fresh image-generation requests.

## Architecture and verification

- Removed the duplicate route-decision module and documented the single-contract boundary in `docs/architecture.md`.
- Added focused task-context boundary tests and updated routing, browser-contract, and regression coverage.
- Verified the complete project check suite before release.
- Docker deployments can upgrade to the `1.6.0` or `v1.6.0` image tag.
