# ChatUI v1.6.6

## Message edit and resend reliability

- Fixes a JavaScript temporal-dead-zone error that could appear when an edited message was routed to image generation.
- Ensures an edited image task that is reclassified as chat receives a new durable chat recovery identifier.
- Creates a fresh client chat job identifier when browser IndexedDB, pending-submit data, or transient display state has been cleared, instead of failing with a recovery-identifier error.

## Recovery behavior

- Decouples chat job identifier creation from transient message-display records and stale local model checks.
- Keeps the root fallback submit workflow aligned with the modular browser workflow.
- Bumps browser asset cache versions so clients load the repaired submit and image workflows.

## Verification

- Added regression coverage for image-generation variable shadowing, image-to-chat rerouting, and missing local recovery state.
- Verified the complete project check suite with 318 passing tests.
- Docker deployments can upgrade to the `1.6.6` or `v1.6.6` image tag.
