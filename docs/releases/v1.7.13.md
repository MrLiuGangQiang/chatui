# ChatUI v1.7.13

Released on July 21, 2026.

## Fixes

- Fixed the root cause of completed chat, image generation, and image editing tasks leaving the composer stuck in the running/stop state.
- Made the upstream interface completion response the authoritative normal completion signal: after the canonical assistant result is persisted and the managed job is cleared, the matching session task is immediately committed as completed.
- Bound completion events to `sessionId`, `submissionId`, `jobId`, and `jobKind` so late callbacks from older tasks cannot release or overwrite a newer task.
- Immediately released the completed run owner and refreshed the composer controls, restoring `发送` without waiting for the outer submit workflow or an exception/finally path.
- Prevented terminal tasks from being resurrected by late recovery events.
- Refreshed browser and static-bundle cache versions so deployed clients receive the lifecycle fix immediately.

## Verification

- Added regression coverage for interface-owned completion ordering, completion identity validation, idempotent duplicate completion, stale-task isolation, terminal recovery protection, and immediate composer projection.
- `npm run check` passes with 391 automated tests.
