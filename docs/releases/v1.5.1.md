# ChatUI v1.5.1

## Bug fixes

- Fixed a route-dispatch argument mismatch that could leave a conversation stuck at the intent-recognition stage after the routing model returned.
- Follow-up requests such as revising a previously generated HTML page now pass the correct request headers and route context, then continue to the configured chat model as expected.
- Applied the fix to both normal messages and quoted-message routing paths.
- Bumped the submit-workflow browser cache version so refreshed clients receive the corrected workflow immediately.

## Engineering

- Added a focused regression test that verifies route headers and route context cannot be shifted by the submit-workflow wrapper.
- Removed the push and pull-request CI workflow. Docker image builds and GitHub Release publishing continue to run only for semantic release tags (`vMAJOR.MINOR.PATCH`).

## Upgrade notes

- No data migration or configuration change is required.
- Refresh the browser after upgrading to ensure the updated submit workflow is loaded. If the application runs from a Docker image, upgrade to the `1.5.1` or `v1.5.1` image tag.
