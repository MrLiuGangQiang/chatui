# ChatUI v1.5.2

## Reliability fixes

- Fixed recovery of in-progress managed chat tasks when switching conversations or refreshing the page.
- Durable chat-job records now remain authoritative when a pending display snapshot lags behind or carries a different job ID, preserving the request payload required to reconnect or recreate the server-side job.
- Pending output display records are reconciled to the durable task before stale-display cleanup, so the visible streaming anchor is not removed during restore.
- Returning to a conversation whose stream is still running in the current page now rebinds the existing stream instead of opening a duplicate recovery connection.
- Local-storage quota failures no longer erase the previous resumable job backup.

## Reasoning history

- Preserved completed reasoning content after refresh even when reasoning is disabled for subsequent requests.

## Engineering

- Added regression coverage for durable-job/display reconciliation, session-switch live-stream rebinding, pending-display retention, reasoning-history persistence, and quota-failure recovery.

## Upgrade notes

- No migration or configuration change is required.
- Refresh the browser after upgrading so the updated browser workflow assets are loaded.
- Docker deployments can upgrade to the `1.5.2` or `v1.5.2` image tag.
