# ChatUI v1.7.24

## 用户可见改进

### 网页预览支持独立新窗口

- AI 回复中识别到完整 HTML 页面时，点击预览图标会优先在独立新窗口打开，不再受聊天界面预览区域尺寸限制。
- 新窗口使用全尺寸预览容器，便于查看页面布局、交互表单和响应式内容。
- 如果浏览器或企业策略拦截弹窗，系统会自动回退到原有的页内预览，不会丢失预览能力。
- 预览按钮的提示文本已更新为“在新窗口打开网页”。

### 预览安全边界保持不变

- 新窗口中的生成网页仍运行在沙箱 iframe 内，支持脚本、表单、下载和弹窗等预览所需能力。
- 预览窗口会主动断开与主应用的 `opener` 引用，避免生成页面反向访问主应用窗口。
- 页面源码通过安全序列化写入预览容器，避免 HTML 内容破坏预览窗口外壳。

## 智能意图路由执行地图同步

- 更新 12 节点执行地图，使说明与当前 `task_contract.v3` 路由实现一致。
- 明确备用模型仅在主路由失败且聊天模型与主路由模型不同时启用。
- 合同校验节点现在说明：结果要么派生为执行计划，要么进入安全澄清；无效模型输出不再被误画为错误终点。
- 澄清节点覆盖模型主动澄清和主、备用模型均返回无效契约两种场景；系统会保存待补充上下文，等待用户继续说明。
- 将“任务完成”改为“任务已交接 / 等待结果或恢复”，准确反映持久化交接后仍可能异步完成、失败或恢复的生命周期。

## 质量验证

- 增加网页预览新窗口、沙箱隔离、无 opener 关联以及弹窗成功时不打开页内对话框的自动化覆盖。
- 更新执行地图回归断言，验证安全澄清和异步恢复文案。
- 完整执行 `npm run check`：项目检查、架构检查、语法检查和全部自动化测试均通过（432 项）。
