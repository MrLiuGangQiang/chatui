# ChatUI v1.5.4

## Session history recovery

- Fixed a storage-transition regression where an incomplete IndexedDB v2 snapshot could hide the fuller conversation history still retained in legacy local storage.
- Partial snapshots now recover missing legacy messages while preserving messages sent after the upgrade and reindexing the combined conversation without collisions.
- Repaired histories are written back as a newer durable v2 snapshot so subsequent loads use the recovered canonical record.
- Complete current snapshots remain authoritative over shorter stale backups.

## Browser delivery

- Bumped the session-persistence and static-bundle cache versions so browsers load the recovery fix immediately after deployment.

## Verification

- Added regression coverage for disjoint transition histories, prefix-only snapshots, canonical reindexing, metadata preservation, and current-snapshot precedence.

## Upgrade notes

- Do not clear browser site data before upgrading if affected conversations need recovery; the retained legacy local-storage backup is the recovery source.
- After upgrading, refresh the page once. Affected histories are repaired automatically when their sessions load.
- Docker deployments can upgrade to the `1.5.4` or `v1.5.4` image tag.
