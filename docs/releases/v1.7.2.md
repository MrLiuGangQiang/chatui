# ChatUI v1.7.2

## Highlights

- Treats multi-image composition as resource selection plus image upload, rather than formatting it as a local image patch.
- Sends the user's current composition request to the image API unchanged, without adding a patch baseline, resource descriptions, duplicated structured changes, or extra execution requirements.
- Keeps semantic history matching responsible only for locating the requested images; unrelated historical images remain excluded from the repeated `image[]` multipart fields.
- Asks for clarification only when the requested images cannot be identified uniquely.

## Validation

- Full project checks and 356 automated tests.
- End-to-end smoke test verifies semantic history selection, exact prompt passthrough, two repeated `image[]` multipart fields, and exclusion of unrelated images at a mock `/v1/images/edits` upstream.
- Headless Microsoft Edge verification against the built static bundle confirms cat and cow are selected from three candidates, the user's prompt is preserved byte-for-byte at the JavaScript string level, and no page errors occur.
